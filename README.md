🚀 Casper Sniper Bot (BingX 선물 자동매매 시스템)

본 프로젝트는 미국 장 개장 직후의 유동성을 활용하는 **'캐스퍼(Casper) 단기 돌파 전략'**을 기반으로 FVG(Fair Value Gap) 필터와 지정가 되돌림(Limit Retest) 기법을 결합하여 개발된 BingX 전용 무인 자동매매 봇입니다.

🧠 핵심 매매 로직

본 시스템은 단순한 시장가 추격 매수가 아닌, **"손익비 극대화와 슬리피지 제로(0%)"**를 목표로 하는 기관급 스나이핑 로직으로 구동됩니다.

기준선 설정 (09:30 ~ 09:35 EST): 미국 본장 개장 직후 첫 5분봉의 고점(High)과 저점(Low)을 당일 돌파 판단을 위한 유동성 기준선으로 캡처합니다. (서머타임 자동 계산 적용)

돌파 및 FVG 필터 감시 (1분봉): 09:35 이후 1분봉이 기준선을 돌파할 때, 단순 돌파가 아닌 **연속된 3개의 캔들 사이에서 꼬리가 겹치지 않는 FVG(불균형 갭)**가 동반되었는지 확인하여 가짜 돌파(Whipsaw)를 걸러냅니다.

지정가 되돌림 예약 (Limit Retest):
조건이 만족되면 즉시 추격(시장가)하지 않고, 돌파된 기준선 가격(EP)에 **지정가(Limit) 주문과 SL(손절)/TP(익절)**를 예약해 둡니다. (손익비 1:2 고정)

타임아웃 자동 취소 (Auto-Cancel):
예약 후 45분 이내에 가격이 되돌아오지 않고 날아가 버리면(눌림목 미발생), 봇이 스스로 낡은 주문을 취소하고 당일 매매를 종료합니다.

🛡️ 시스템 주요 방어 기제

거시경제 이벤트 필터링: 당일 CPI, FOMC, NFP 등 주요 미국 경제 지표 발표가 있는 날은 봇이 스스로 판단하여 하루 종일 매매를 쉬고 관망합니다.

최소 손절폭(minRiskGap) 보정: 횡보장에서 캔들이 너무 수축되어 있을 때, 기본 스프레드만으로 손절이 나가는 것을 막기 위해 현재가의 최소 0.1%를 강제 손절폭으로 확보합니다.

텔레그램(Telegram) 실시간 알림: 이벤트 데이 감지, 예약 주문 세팅, 체결 완료, 타임아웃 취소 등 모든 중요 액션을 스마트폰으로 실시간 전송합니다.

📊 백테스트 인사이트 (BTC 페어)

100일 시뮬레이션 결과, 본 봇의 1분봉 기반 타이트한 지정가 로직은 호가창이 두껍고 유동성이 풍부한 **BTC(비트코인)**에 가장 최적화되어 있습니다.

🟢 정직한 되돌림: 유동성이 풍부해 FVG 생성 후 되돌림(Retest)을 주는 빈도가 매우 정직합니다.

🟢 안정적인 방어력: 꼬리(Wick)로 불필요하게 손절선(SL)을 터치하고 올라가는 악성 휩소가 적어 매우 안정적인 승률(약 53%+)과 뛰어난 최대 낙폭 방어력(MDD -5%대)을 기록합니다.

본 시스템은 이러한 특성을 극대화하기 위해 BTC 단일 종목 집중 운영을 기본으로 권장합니다.

⚙️ 설치 및 세팅 가이드

1. 사전 요구사항

Node.js (v16 이상 권장)

BingX API Key (선물 거래 권한 활성화)

Financial Modeling Prep (FMP) 무료 API Key (거시경제 필터용)

2. 패키지 설치

프로젝트 폴더를 열고 아래 명령어로 필수 라이브러리를 설치합니다.

npm install axios dotenv date-fns date-fns-tz
npm install -D typescript ts-node @types/node


3. 환경 변수 설정 (.env)

프로젝트 최상단에 .env 파일을 생성하고 아래 양식에 맞게 입력합니다.

# BingX API 설정
BINGX_API_KEY=당신의_빙엑스_API_키
BINGX_API_SECRET=당신의_빙엑스_시크릿_키

# 봇 트레이딩 설정
SYMBOL=BTC-USDT
LEVERAGE=10
TRADE_AMOUNT_USDT=100

# API 기능 설정
ECONOMIC_API_KEY=FMP_무료_API_키

# 텔레그램 알림 설정 (선택사항)
TELEGRAM_BOT_TOKEN=텔레그램_봇_토큰
TELEGRAM_CHAT_ID=수신받을_채팅방_아이디


🚀 실행 방법

터미널을 열고 프로젝트 폴더 경로에서 아래 명령어를 입력하여 봇을 실행합니다.

npx ts-node index.ts


(참고: 리눅스/서버 환경에서 터미널을 종료해도 봇이 계속 백그라운드에서 동작하게 하려면 nohup npx ts-node index.ts > output.log 2>&1 & 명령어를 사용할 수 있습니다.)
